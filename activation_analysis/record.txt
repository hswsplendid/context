│ - Identity test：原文 vs 原文（完全相同），所有层 cosine 应 = 1.0                                     │
│ - Minimal perturbation：只改 1 个词，浅层 cosine 应 > 0.95                                            │

│ - 13 层采样，每次实验 2 次前向传播                                                                    │
│ - 收集 token 级 + segment 级指标（cosine, L2, CKA）                                                   │
│                                                                                                       │
│ Phase 3: Type 2 主实验（语义改写）                                                                    │
│ - 2 个 Agent 模板 × 3 个替换对 × 4 个上下文长度 × 3 个扰动比例 × 3 个位置 = 216 次实验                │
│ - 先用模型生成改写（缓存），再做激活比较                                                              │
│ - 只收集 segment 级指标（cosine_segment, CKA）                                                        │
│                                                                                                       │
│ Phase 4: 汇总分析 + 可视化                                                                            │
│ - 合并所有 Phase 的 CSV                                                                               │
│ - 生成 5 类标准图 + 跨模板/跨类型对比图                                                               │

│ GPU 分配策略                                                                                          │
│                                                                                                       │
│ Qwen3-30B-A3B（bfloat16）约需 ~60GB 显存。A800 80GB 单卡可容纳但紧张。                                │
│                                                                                                       │
│ 分配逻辑：                                                                                            │
│ - 用户输入 GPU 列表（如 0,3,4,5），共 N 张卡                                                          │
│ - Qwen3-30B-A3B 需要 cards_per_model 张卡（默认 2，可配置 1-4）                                       │
│ - num_workers = N // cards_per_model 个并行 worker                                                    │
│ - 每个 worker 分配连续的 cards_per_model 张卡                                                         │
│ - 实验网格均匀分配到 worker（按 chunk 切分）                                                          │
│                                                                                                       │
│ 示例：                                                                                                │
│ - --gpus 0,1,2,3 + --cards-per-model 2 → 2 workers：[0,1] 和 [2,3]                                    │
│ - --gpus 0,1,2,3,4,5,6,7 + --cards-per-model 2 → 4 workers                                            │
│ - --gpus 0 + --cards-per-model 1 → 1 worker（串行）                                                   │
│                                                                                                       │
│ 实现方式：torch.multiprocessing.spawn，每个子进程设置自己的 CUDA_VISIBLE_DEVICES，独立加载模型。      │

│ 7. generate_analysis_report()：Phase 4，汇总统计 + 全部可视化 + 假设验证总结                          │
│ 8. main()：编排所有 phase                                                                             │

│ assert shallow_layer_cosine > 0.95                                                                    │

│                                                                                                       │

│ - Worker 异常退出 → 主进程记录错误，其他 worker 继续                                                  │
│ - 部分结果可恢复 → 检查已有 CSV，跳过已完成的实验                                                     │
